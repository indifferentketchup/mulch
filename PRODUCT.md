# Product

## Register

product

## Users

Game-server operators. They are often mid-incident — a server crashed, a mod is misbehaving, a player is reporting problems — and they need to paste a log and get to diagnosis fast. They share short URLs with mod developers or community helpers for collaborative debugging. Technical users who live in the command line but appreciate a clean, fast web interface when their server is on fire.

## Product Purpose

iblogs is a paste-and-share tool for game-server logs (Minecraft, Hytale, Project Zomboid). It strips PII, assigns a short URL, renders with syntax highlighting and line numbers, and runs automated analysis to surface problems. Success is measured by how fast someone can go from a crash report to a diagnosis they can act on.

## Brand Personality

Hacker-friendly, playful, dark. The voice is irreverent (the project lives under "indifferent ketchup") but the tool is dead serious about its job. Like a well-configured terminal — functional beauty, not decoration. Monospace culture, inside jokes, fast feedback. Never at the expense of clarity.

## Anti-references

- AI-generated SaaS landing pages: no gradient text, no blur/glassmorphism cards, no pastel palettes, no eyebrow text above every section, no hero-metric templates
- Game-themed interfaces: the tool analyzes game logs but is not the game itself — no Minecraft pixel art, no fantasy UI chrome
- Hollywood hacker cliches: no green-on-black matrix terminal pastiche, no "cyber" fonts, no cheesy retro-computing tropes
- Over-designed dashboards: no widgets, sparklines, or heavy admin-panel chrome. The log is the hero

## Design Principles

1. **Information first** — the log is the hero. Chrome, navigation, and decoration step back. Every pixel of chrome must earn its place.
2. **Speed over style** — every interaction should feel instant. Perception of speed matters more than visual flourish.
3. **Clarity in the dark** — dark theme is functional (logs are read at night, mid-incident). Contrast and readability are non-negotiable. Typography does the heavy lifting.
4. **Playful, not childish** — the brand has personality but never undermines the tool's credibility. Terminal aesthetics, not game aesthetics. Wit, not whimsy.
5. **Stay out of the way** — the tool disappears when it's doing its job. Users should forget the interface exists and focus entirely on their log.

## Accessibility & Inclusion

WCAG 2.2 AA. Dark theme must maintain contrast (≥4.5:1 body text, ≥3:1 large text) across all severity and state variants. Reduced motion respected on all animations and transitions. Full keyboard navigation through settings, problem panels, and log interaction. Screen-reader labels on interactive controls and the log content area.
